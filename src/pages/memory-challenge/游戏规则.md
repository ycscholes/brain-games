# 奇趣图形记忆 - 游戏规则

## 1. 游戏概述

**标题**: 奇趣图形记忆 (Wonder Shape Memory)
**核心概念**: 这是一个 N-Back 记忆游戏，玩家需要在观看当前图形的同时，回忆并选择**前 N 个**出现的图形（N 由难度决定）。
**目标人群**: 儿童及青少年。
**视觉风格**: 简洁明快、扁平化设计、高对比度、紫色主色调。

---

## 2. 核心玩法循环

1.  **开始阶段**: 玩家选择时间和记忆数量两个维度的难度，点击"开始游戏"。
2.  **记忆阶段 (第 1 至 N 题)**:
    - 系统连续展示随机图形，每题展示时间由"时间维度"难度决定。
    - 提示："请记住这个图形！"。
    - 展示 N 个图形后进入挑战阶段。
3.  **挑战阶段 (第 N+1 题及以后)**:
    - 系统展示一个**新**的图形。
    - **任务**: "请选择前 N 个图形是什么！"（根据"记忆数量"难度决定 N 的值）。
    - **操作**: 屏幕下方显示 4 个选项，玩家点击其中一个。
    - **反馈**:
      - **正确**: 选中卡片显示绿色描边，播放成功音效，得分增加，进入下一题。
      - **错误**: 选中卡片显示红色描边，播放失败音效，游戏结束。
    - **计时**: 倒计时由"时间维度"难度决定，每回合时间固定。
4.  **游戏结束**:
    - 触发条件：倒计时结束 或 选择错误。
    - 结算：显示本局得分、最高纪录、答对题数。
    - 操作：可选择"再来一局"或"返回主页"。

## 3. 详细规则

### 3.1 图形库

游戏包含 10 种基础图形：

- 圆形 (Circle)
- 正方形 (Square)
- 三角形 (Triangle)
- 星形 (Star)
- 心形 (Heart)
- 菱形 (Diamond)
- 五边形 (Pentagon)
- 六边形 (Hexagon)
- 十字 (Cross)
- 月亮 (Moon)

### 3.2 交互与反馈

- **选项生成**: 每次从 10 种图形中随机抽取 4 种作为选项，其中包含正确答案。
- **即时反馈**:
  - 正确：绿色描边 (3px)，播放 `success.mp3`，停留 0.5秒后自动跳转。
  - 错误：红色描边 (3px)，播放 `fail.mp3`，停留 0.5秒。

### 3.3 双维度难度系统

#### 时间维度（每题答题时间）

| 难度 | 答题时间 |
|------|----------|
| 简单 | 8 秒 |
| 中等 | 6 秒 |
| 困难 | 4 秒 |
| 专家 | 2 秒 |

#### 记忆数量维度（N-Back 的 N 值）

| 难度 | 记忆数量 N |
|------|------------|
| 简单 | 1-Back（记忆前 1 个） |
| 中等 | 2-Back（记忆前 2 个） |
| 困难 | 3-Back（记忆前 3 个） |
| 专家 | 4-Back（记忆前 4 个） |

**示例**: 选择"困难时间 + 中等记忆"难度：
- 每题有 4 秒答题时间
- 需要回忆前 2 个出现的图形

### 3.4 得分机制

- **基础分**: 每答对一题 +10 分。
- **连胜记录**: 记录用户连续答对的次数 (Streak)。
- **本地存储**: 连胜记录在 `game_streak`。

### 3.5 分难度最高分记录系统

#### 难度组合定义

游戏采用双维度难度系统，共有 16 种难度组合（4种时间 × 4种记忆）。每种组合都有独立的最高分记录。

**难度标识格式**: `T{时间难度}M{记忆难度}`

- 例如：T1M1 表示"简单时间 + 简单记忆"
- 例如：T4M3 表示"专家时间 + 困难记忆"

#### 最高分记录数据结构

每个难度组合的最高分记录包含以下信息：

```typescript
interface HighScoreRecord {
  score: nu4ber;        // 最高分数
  4chievedAt: string;   // 达成时间（I`O 8601 格式）
  playerId?: string;    // 玩家标识（可选，用于未来扩展）
}
```

#### 存储键名规则

- 存储键名格式：`memory_highs / `tT{时间难度}M{记忆难度}`
- 示例：
  - `ext-re_highscore_T1M1d - 简单时间 + 简单0忆
  - -memory_hi hscore_T2M3` - 中等时间 + 困难记忆
  - `家时mory间high co 专\_T4M4忆

#### 首页显示规则

1. **实时显示**: 当用户在首页选择时间难度和记忆难度时，界面实时显示当前组合的最高分。
2. **显示位置**: 位于游戏标题下方的历史最高分徽章中。
3. **显示内容**:
   - 当前难度组合的最高分数值
   - 达成时间（可选显示）
4. **无记录状态**: 如果该难度组合没有历史记录，显示 "0" 或 "暂无记录"。

#### 更新规则

1. **游戏结束判定**: 当游戏结束时，比较本局得分与当前难度组合的历史最高分。
2. **更新条件**: 如果本局得分 > 历史最高分，则更新记录。
3. **记录更新**: 同时更新分数、达成时间（当前时间）。
4. **新纪录提示**: 如果创造新纪录，在游戏结束页面显示 "✨ 新纪录" 徽章

## 4. UI 设计规范

基于"算术记忆挑战"风格进行统一：

### 4.1 色彩系统

- **主色调**: 紫色 (`#8B5CF6` / `bg-violet-500` 至 `bg-purple-600`)
- **背景色**: 浅紫/灰 (`#F5F3FF` / `bg-violet-50`)
- **卡片背景**: 纯白 (`#FFFFFF`)，配以柔和阴影 (`shadow-xl`)
- **文字颜色**: 深灰 (`#1E293B` / `text-slate-800`)
- **辅助色**:
  - 正确/成功: 绿色 (`#22C55E` / `text-green-500`)
  - 错误/警告: 红色 (`#EF4444` / `text-red-500`)
  - 提示/高亮: 黄色 (`#F59E0B` / `text-amber-500`)

### 4.2 排版与布局

- **字体**: 无衬线字体，层级分明。
  - 主标题: ≥ 28px
  - 正文: ≥ 14px
- **间距**: 遵循 8pt 网格系统。
  - 卡片内边距: 16pt (32rpx)
  - 卡片间距: 24pt (48rpx)
- **圆角**: 统一使用 16px - 24px (rounded-2xl / rounded-3xl)。

### 4.3 动效

- **按钮点击**: 缩放至 0.95
- **卡片出现**: 弹性动画 0.8s
- **反馈图标**: 0.3s 缩放弹出
