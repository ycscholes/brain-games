# 多任务处理

## 训练目标

通过持续主轨校准和短时插入判断，训练注意分配、任务切换与冲突抑制。

## 入口与状态

- 路由：`/pages/dual-task/index`
- `gameId`：`dual-task`
- 记录模式：`command-center`
- 状态：开始、60 秒指挥台、结算

## 完整玩法

玩家持续观察往返移动的主轨光标，在光标进入目标区时点击“校准”。同时页面会弹出奇偶、是否大于 50、颜色、方向或 Stroop 插入任务，需要在时限内选择答案。同一个插入窗口中同时完成主轨命中与插入答对会获得同步奖励。

一局分为热身（0-15 秒）、干扰（15-45 秒）和冲刺（45-60 秒）。后半程主轨更快、插入更频繁，并逐步加入 Stroop 冲突。

## 模式与难度

普通模式目标区宽 22%，插入任务冲刺时限 2 秒；困难模式目标区宽 16%，冲刺时限 1.4 秒。困难模式的主轨周期更短、插入间隔更短、Stroop 比例更高。

## 游戏内计分

- 主轨命中：1 分
- 插入任务答对：2 分
- 同窗口双线完成：额外 1 分
- 每达到 5 连击：额外 1 分
- 错误、漏答或超时：不扣分，但连击归零

连续 3 次错误会进入 2 秒恢复窗口。最终游戏分数封顶 40。

## 积分管线

基础转换率为 1.0。普通和困难分别交给共享难度倍率与封顶逻辑处理，页面不自行计算奖励倍率。

## 训练记录

结算记录最终分数、积分、`mode: "command-center"`、难度和实际时长。普通与困难分别保存最高分。

## 实现与测试

`gameLogic.ts` 负责阶段、轨道位置、插入题、答案判断、计分和恢复条件；`index.tsx` 负责定时器、交互、渲染和存储。`tests/unit/dualTaskGameLogic.test.ts` 覆盖核心纯逻辑。
