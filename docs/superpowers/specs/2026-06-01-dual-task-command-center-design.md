# 多任务处理重构设计：双线指挥台

## 背景

当前 `dual-task` 已有交替、同时、Stroop 三种模式，但主体验偏向两张答题卡并排刷题。玩家的压力主要来自倒计时、按钮数量和题面变化，不够像在“分配注意力”。本次重构目标是把游戏改成一个持续运行的主任务叠加短促插入任务，让玩家明确感到自己在同时维持稳定操作和处理干扰。

参考方向：

- NeuroRacer：持续主任务加目标信号识别，强调多任务训练中的持续控制和插入干扰。
- Dual N-Back：双信息流持续推进，训练持续更新而不是孤立答题。
- Stroop：用冲突信息训练抑制控制，适合作为后半局干扰规则。
- Bop It / WarioWare：短指令、即时反馈、节奏逐步加快，增强再来一局的动机。

## 目标

- 保留内部 `gameId: "dual-task"`、路由 `/pages/dual-task/index`、积分转换率和训练记录兼容。
- 将核心玩法从“并排答题卡”重构为“主线稳定 + 插入判断”的双线指挥台。
- 普通好表现保持约 20-40 分，困难好表现约 30-45 分，继续交给 `getAwardedPoints("dual-task", score, rewardDifficulty)` 统一换算积分。
- 降低挫败感：错误主要断连击，不做重罚；连续错误后给短恢复窗口。
- 让玩法一眼可懂：玩家一直盯住轨道，同时处理弹出的短指令。

## 非目标

- 不改变宠物积分经济的基础规则。
- 不迁移旧训练记录。
- 不实现完整 Dual N-Back。
- 不保留三个旧模式作为并列主入口；Stroop 会被吸收为后半局干扰类型。

## 核心玩法

游戏名可以继续显示为“多任务处理”，玩法副标题改为“双线指挥台”。

一局 60 秒，玩家同时处理两条压力线：

1. 主线稳定任务
   - 屏幕中央有一条横向轨道和移动光标。
   - 光标进入目标区时，玩家点击“校准”。
   - 命中会增加稳定度和分数；过早、过晚或漏掉会断连击。

2. 插入判断任务
   - 轨道上方短暂弹出指令卡。
   - 指令卡持续 1.2-2.5 秒，要求玩家在 2-4 个大按钮中选择。
   - 指令类型包括奇偶、大小、颜色、方向、Stroop 颜色词冲突。

3. 双线奖励
   - 如果玩家在同一个插入窗口内完成一次主线命中和一次插入正确，获得额外“同步奖励”。
   - 这是核心爽点：玩家能感觉到自己真的兼顾了两件事。

## 阶段节奏

一局固定分三段：

| 阶段 | 时间 | 行为 |
| --- | --- | --- |
| 热身 | 0-15s | 主线速度低，插入频率低，不出现 Stroop |
| 干扰 | 15-45s | 插入频率提高，开始出现方向和颜色冲突 |
| 冲刺 | 45-60s | 主线速度提高，Stroop 比例提高，插入窗口更短 |

难度只影响节奏参数，不改变基本规则：

| 难度 | 主线速度 | 插入间隔 | 插入时限 | Stroop 比例 | 积分难度 |
| --- | --- | --- | --- | --- | --- |
| 普通 | 慢到中 | 3.0-4.5s | 2.0-2.5s | 低 | normal |
| 困难 | 中到快 | 2.0-3.2s | 1.4-2.0s | 中高 | hard |

旧的 easy/expert 配置不继续作为主入口。开始页只保留“普通 / 困难”，减少选择负担。

## 计分

游戏内分数保持低尺度，避免过早撞积分封顶。

- 主线命中：+1
- 插入判断正确：+2
- 同步奖励：+1
- 每 5 连击额外 +1
- 错误、漏答、超时：不扣分，断连击
- 连续 3 次错误：触发 2 秒恢复窗口，插入频率降低，主线速度暂缓

结算记录：

```ts
recordTrainingSession({
  gameId: "dual-task",
  score: finalScore,
  awardedPoints,
  mode: "command-center",
  difficulty: rewardDifficulty,
  durationSeconds,
  outcome: "completed",
})
```

积分必须继续使用：

```ts
const awardedPoints = getAwardedPoints("dual-task", finalScore, rewardDifficulty);
addPointsToPet("dual-task", finalScore, rewardDifficulty);
```

页面不得自行乘倍率或手动封顶。

## 任务类型

### 主线任务

主线只做一件事：在光标经过目标区时点击校准。

状态：

- `idle`：等待下一次目标窗口
- `active`：目标窗口可命中
- `hit`：命中反馈
- `miss`：错过反馈

判定：

- 光标中心在目标区内：命中
- 光标中心刚离开目标区且未点击：漏判
- 光标在目标区外点击：误触

### 插入任务

插入任务使用短题面和固定大按钮，避免阅读负担过高。

| 类型 | 示例 | 选项 |
| --- | --- | --- |
| 奇偶 | `17` | 奇 / 偶 |
| 大小 | `42 > 50?` | 是 / 否 |
| 颜色 | `蓝色块` | 红 / 蓝 / 黄 / 绿 |
| 方向 | `←` | 左 / 右 |
| Stroop | 红字显示“蓝” | 按“字色”或“字义” |

Stroop 只在干扰和冲刺阶段出现，且题面必须明确显示本轮目标，例如“按字色”。

## UI 结构

开始页：

- 标题：多任务处理
- 副标题：稳定主线，同时处理插入指令
- 两个难度卡：普通、困难
- 当前最高分、训练目标说明

游戏页：

- 顶部：剩余时间、分数、连击
- 中央：主线轨道、移动光标、目标区、稳定度
- 上方浮层：插入指令卡
- 底部：2-4 个大按钮，按钮随插入任务更新
- 反馈：命中、漏判、插入正确、同步奖励用短动效提示

结果页：

- 总分
- 稳定命中率
- 插入正确率
- 同步奖励次数
- 最长连击
- 获得积分

## 测试计划

新增或更新单元测试：

- 普通/困难阶段参数正确。
- 阶段随 elapsedMs 从热身进入干扰和冲刺。
- 主线命中、误触、漏判判定正确。
- 插入任务答案判断正确。
- 同步奖励只在同一窗口内主线和插入都正确时触发。
- 连续错误触发恢复窗口。
- 典型普通好表现落在 20-40 分。
- 典型困难好表现落在 30-45 分。
- 结算仍通过 `getAwardedPoints("dual-task", finalScore, rewardDifficulty)`。

## 迁移策略

- 第一阶段只替换 `dual-task` 页面内部玩法。
- 历史最高分 key 可以继续使用 `dual_task_best_command-center`，旧模式最高分不迁移。
- 训练记录继续使用 `gameId: "dual-task"`。
- 首页和分享文案可以保留“多任务处理”，降低改动范围。

## 风险与缓解

- 风险：主线轨道在小屏上太紧张。
  - 缓解：目标区宽度随屏幕比例设置下限，普通难度增加命中窗口。

- 风险：插入任务遮挡主线。
  - 缓解：插入卡固定在轨道上方，不覆盖目标区。

- 风险：玩法比旧版更依赖动画时间。
  - 缓解：逻辑层用 elapsedMs 计算状态，测试不依赖真实定时器。

- 风险：玩家刚开始不理解同步奖励。
  - 缓解：首页只讲“校准轨道 + 处理弹窗”，同步奖励在结果页和反馈中自然出现。

## 产品决策

- 显示名继续叫“多任务处理”，避免首页和训练记录产生断裂；玩法副标题使用“双线指挥台”。
- 不保留旧三个模式入口，第一版专注一个主循环，减少实现和测试范围。
