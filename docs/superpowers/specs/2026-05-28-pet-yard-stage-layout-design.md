# 宠物小院全屏舞台布局设计

## 背景

宠物小院已完成绘本游戏风换肤、互动反馈和本地生成资产接入，但当前页面仍保留较多卡片和管理面板结构，例如统计卡、详情卡、商店卡、领养面板等。用户希望继续优化为更偏游戏的呈现：去掉卡片展现形式，强调当前宠物，弱化宠物数据，并把领养流程收进弹窗。

## 决策记录

- 主布局：采用全屏小院舞台，而不是卡片混合布局或双区面板。
- 主页面信息：保留宠物名、简短状态、饱食度、积分余额。
- 主页面移除：宠物总数、存活数量、下次领养卡、创建时间、最后更新、共享积分详情、离开宠物统计等管理型信息。
- 领养流程：只在主页面保留入口，具体选择和命名流程使用居中弹窗。
- 宠物切换：主页面不常驻宠物列表，点击当前宠物或名字打开居中选择弹窗。
- 图片显示：所有宠物和食物图片必须保持原始比例，不允许拉伸。

## 目标

1. 打开宠物小院时，第一视觉是小院舞台和当前宠物，而不是数据面板。
2. 主页面只保留照顾当前宠物需要的最小信息：名字、状态、饱食度、积分余额和操作入口。
3. 领养和宠物切换使用弹窗承载，保持主页面干净。
4. 宠物和食物图片使用等比显示，避免变形。
5. 不改变现有宠物养成规则、积分规则、同步规则和资产路径策略。

## 非目标

- 不新增养成系统、任务系统、装饰系统或背包系统。
- 不新增领养独立页面。
- 不调整领养成本、喂食成本、饱食度衰减或死亡规则。
- 不重新生成图片资产。
- 不重构首页或其他训练页面。

## 用户体验设计

### 主页面

主页面改为全屏小院舞台：

- 页面背景和主舞台合成一个连续场景，避免多个独立卡片堆叠。
- 当前宠物在舞台中央大尺寸展示。
- 宠物气泡显示当前台词，例如等待、饥饿、吃饱、被抚摸等。
- 顶部或角落放轻量 HUD，显示：
  - 当前宠物名
  - 简短状态：状态稳定、需要喂食、已离开
  - 饱食度
  - 当前积分余额
- 底部或舞台边缘放游戏化操作入口：
  - 喂食
  - 抚摸
  - 选择宠物
  - 领养

主页面不展示管理型明细：

- 宠物总数
- 存活数量
- 下次领养统计卡
- 创建时间
- 最后更新
- 共享积分详情卡
- 离开宠物统计

### 领养弹窗

点击领养入口后打开居中游戏弹窗：

- 背景遮罩覆盖舞台。
- 弹窗包含外观横滑选择、名字输入、费用提示和确认按钮。
- 第一只宠物免费时显示免费提示；后续领养显示本次积分成本。
- 弹窗提供明确关闭按钮。
- 弹窗内容高度受控，移动端不能撑出屏幕；外观列表可横向滚动。

### 宠物选择弹窗

点击当前宠物、宠物名或“选择宠物”入口后打开居中游戏弹窗：

- 展示已有宠物的头像、名字和简短状态。
- 当前宠物高亮。
- 点击其他宠物后切换 activePetId，关闭弹窗，并播放切换反馈。
- 已离开宠物可选择查看，但状态表达必须明显。

### 喂食与抚摸

- 喂食入口可以直接显示食物选择，也可以先打开小型食物选择层；实现时优先复用现有食物列表逻辑，避免引入新数据结构。
- 喂食成功后保留现有食物浮层、饱食度变化和台词变化。
- 抚摸当前宠物保留 `cuddle` 动作和亲密台词。
- 已离开宠物不触发喂食和正向抚摸反馈。

## 图片比例要求

- `PetSprite` 内的宠物图片必须使用等比显示，例如 `mode="aspectFit"` 或等效 CSS `object-fit: contain`。
- 食物图标同样使用等比显示。
- 固定展示区域可以裁切外部布局空间，但不能拉伸图片本体。
- 小头像、主舞台大图、食物按钮都必须满足该规则。

## 技术设计

### 主要改动区域

- `src/pages/pet/index.tsx`
  - 将 hero、统计、详情、商店、领养面板重组为舞台视图。
  - 新增领养弹窗状态。
  - 新增宠物选择弹窗状态。
  - 移除主页面管理型字段渲染。
- `src/pages/pet/index.scss`
  - 删除或弱化卡片式布局样式。
  - 新增全屏小院舞台、HUD、操作栏、居中弹窗样式。
  - 确保移动端弹窗和舞台布局不重叠。
- `src/pages/pet/components/PetSprite/index.tsx`
  - 将图片模式从拉伸填充改为等比显示。
- `src/pages/pet/components/PetSprite/index.scss`
  - 确保主图、小头像、fallback 的尺寸稳定，图片不变形。

### 数据流

数据流保持现状：

1. 页面读取 `PetStorageData`。
2. `syncPetData` 负责刷新宠物状态。
3. `feedPet` 负责喂食和扣积分。
4. `adoptPet` 负责领养和扣积分。
5. 切换宠物只更新 `activePetId`。
6. 弹窗开关、反馈台词、喂食浮层都是本地 UI 状态，不落库。

### 错误处理

- 图片加载失败继续走 emoji fallback。
- 领养名字为空时继续 toast。
- 积分不足时继续 toast。
- 已离开宠物不可喂食，继续 toast 或状态台词提示。
- 弹窗关闭不应清空已存在宠物数据；领养弹窗关闭可清理临时名字输入。

## 验收标准

1. 主页面没有统计卡片和详情卡片堆叠，视觉上是一个完整小院舞台。
2. 当前宠物是页面最突出的元素。
3. 主页面只显示宠物名、简短状态、饱食度、积分余额和操作入口。
4. 领养流程通过居中弹窗完成，主页面不直接展示完整领养表单。
5. 宠物切换通过居中弹窗完成，主页面不常驻宠物列表。
6. 宠物和食物图片等比显示，不被拉伸。
7. 喂食、抚摸、切换、领养、死亡态逻辑与现有行为一致。
8. 移动端下弹窗内容不超出屏幕，文字不重叠。

## 测试计划

- 运行 `npm run typecheck`。
- 运行 `npm run lint`。
- 运行 `npm test -- tests/unit/petStorage.test.ts`。
- 运行 `npm run build:weapp`。
- 运行 `npm run build:h5`。
- H5 或小程序手动验证：
  - 无宠物时只显示领养入口和空小院提示。
  - 有宠物时主页面突出当前宠物。
  - 领养弹窗可打开、关闭、选择外观、输入名字、确认。
  - 宠物选择弹窗可打开、切换、关闭。
  - 喂食成功后积分和饱食度变化，反馈正常。
  - 已离开宠物不可喂食，视觉状态清晰。
  - 图片在主舞台、头像、食物按钮中不变形。
