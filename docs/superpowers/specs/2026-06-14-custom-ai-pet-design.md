# 自定义 AI 宠物设计

## 1. 目标

在宠物小院中增加“AI 自定义宠物”领养方式。用户上传一张真实宠物照片或宠物插画，系统自动识别宠物特征和最接近的现有宠物模板，生成与当前水彩绘本风格一致的四状态宠物套图：

- `idle`
- `feed`
- `cuddle`
- `hungry`

自定义宠物沿用现有宠物的饥饿、喂食、抚摸、成长和离开规则，并出现在首页浮窗、宠物小院、宠物速数、农场进出和宠物记忆等全部宠物场景中。

## 2. 首版范围

### 2.1 用户与价格

- 功能测试通过后直接向全部用户开放，不做白名单灰度。
- 每位用户最多成功生成 1 只自定义宠物。
- 每次生成价格为 300 宠物积分。
- 用户可免费整套重做 1 次，重做成功后替换首次结果，不同时保留两套候选结果。
- 同一用户同时只能存在 1 个活动生成任务。
- 首版不增加业务层 UID 限流、IP 限流或每日总预算限制。
- 仍保留重复请求去重、任务幂等锁和云服务限流后的退避重试，避免重复生成或重复扣费。

### 2.2 上传范围

- 每次上传 1 张图片。
- 支持真实宠物照片或宠物插画。
- 上传前声明用户必须拥有图片的合法使用权。
- 输入应只包含 1 只清晰宠物，主体不能过小或严重遮挡。
- 明显包含人物主体、多只宠物、非宠物内容、格式错误或尺寸不合规时拒绝提交。
- 生成只提取宠物的物种、毛色、花纹、体型和配饰等身份特征，不复刻上传插画的原始画风。

### 2.3 模板映射

AI 自动把输入映射到现有 7 种模板之一：

- 小猫
- 小狗
- 小兔
- 小熊
- 熊猫
- 守宫
- 小乌龟

用户不查看或修正映射结果。现有范围之外的宠物也继续生成，由 AI 自动选择动作骨架和体型最接近的模板。模板只决定动作、食物和游戏兼容行为，界面仍使用用户命名和 AI 识别出的宠物外观，不把范围外宠物直接标记为模板物种。

## 3. 用户流程

### 3.1 提交

1. 用户在“领养新宠物”中选择“AI 自定义宠物”。
2. 用户选择并裁剪一张图片，页面展示主体预览。
3. 页面明确展示：
   - 价格为 300 宠物积分；
   - 每位用户只能成功生成 1 只；
   - 可免费整套重做 1 次；
   - 原图会长期保留；
   - 原图和生成结果仅本人可在应用内访问；
   - 图片将用于 AI 识别与生成。
4. 小程序通过受限上传授权把原图写入用户私有暂存路径。
5. 服务端检查原图、用户资格、余额和活动任务。
6. 服务端在事务中冻结 300 积分、创建任务并记录本次提交锁。
7. 提交成功后用户可离开页面，生成任务继续在后台运行。
8. 页面可在用户主动操作后申请订阅消息权限；订阅消息仅用于完成提醒，不影响结果领取。

暂存上传失败时不创建任务或冻结积分。上传成功但任务事务失败时删除暂存文件；定时清理任务负责删除异常中断产生的孤立暂存文件。

### 3.2 生成中

宠物页展示单一任务状态区域，不要求用户停留：

- 正在识别宠物
- 正在绘制默认形象
- 正在生成互动状态
- 正在检查结果

用户离开并再次进入时，从服务端恢复任务状态，不重新上传、不重新冻结积分、不重复启动任务。

### 3.3 预览与领养

四张图片全部完成并通过质量检查后，任务进入永久保留的预览状态。用户可以：

- 查看四状态结果；
- 输入宠物名字并确认领养；
- 免费整套重做 1 次；
- 放弃结果并解除积分冻结。

确认领养时，服务端在一个事务中：

1. 创建自定义宠物数据；
2. 将冻结的 300 积分转为实际消费；
3. 验证单次生成资格已由当前任务占用；
4. 将任务标记为已领养。

重做开始后保留第一套结果。第二套全部成功后再原子替换；重做失败时仍可使用第一套结果。

放弃结果时：

- 删除候选结果图；
- 解除 300 积分冻结；
- 原图继续保留；
- 单次生成资格不恢复。

### 3.4 删除

用户可永久删除自定义宠物。二次确认后，服务端通过幂等删除任务清除：

- 原始上传图片；
- 首次和重做产生的全部结果图；
- 自定义宠物数据；
- 生成任务明细和可关联内容。

删除后生成资格不恢复。系统仅保留一个不含图片、宠物名称、识别特征或任务详情的最小额度墓碑，用于记录该账号已使用首版单次生成资格。

## 4. 生成架构

### 4.1 推荐方案

采用 CloudBase 云函数、文档数据库和私有云存储组成的持久化步骤任务，不在客户端串行执行生成流程，也不在首版引入独立 CloudBase Run 队列服务。

组件职责：

- 小程序：图片选择、裁剪、提交、任务状态展示、预览、命名、重做、领养和删除。
- 提交云函数：身份校验、私有上传授权、资格检查、积分冻结和任务创建。
- 异步生成 Worker：内容检查、特征识别、模板映射、四状态生成、透明处理和质量检查。
- 定时恢复任务：扫描因超时、实例终止或临时错误停滞的任务，取得幂等锁后继续下一步骤。
- 查询云函数：只向任务所属用户返回任务状态和私有图片临时 URL。
- 领养与删除云函数：处理事务结算、资源替换和幂等清理。

云函数最长执行时间有限，因此 Worker 每次只推进一个可恢复步骤，并把进度写回数据库。任何步骤可安全重复执行，不依赖某个函数实例持续存活。

### 4.2 身份锚点流水线

生成流程按以下顺序执行：

1. 检查输入格式、主体数量和内容安全。
2. 使用多模态识别模型输出结构化特征：
   - 物种描述；
   - 主色和辅助色；
   - 花纹与明显标记；
   - 体型；
   - 耳朵、尾巴、壳体等结构特征；
   - 配饰；
   - 最接近的现有模板。
3. 结合原图、结构化特征和模板姿态生成 `idle`。
4. 对 `idle` 做身份、构图和素材质量检查。
5. 以 `idle` 为身份锚点，分别派生 `feed`、`cuddle` 和 `hungry`。
6. 对四状态做一致性检查。
7. 处理为透明 PNG，并写入 AI 生成元数据。
8. 四张全部通过后，在事务中写入生成资格墓碑并进入预览。

单一状态失败时只重试该步骤，不重新生成已通过的状态。身份锚点失败时重新生成 `idle`，并使其派生状态失效。

### 4.3 视觉要求

生成结果统一遵循现有宠物素材规范：

- 水彩绘本风格；
- 单只全身宠物居中；
- 1:1 PNG；
- RGBA 透明背景；
- 四周保留安全边距；
- 无场景背景、地面、投影、边框、文字和可见水印；
- 在首页浮窗和游戏小尺寸下仍有清晰轮廓；
- 四状态保持物种、主色、花纹、体型和关键配饰一致。

线上生成流程应复用并扩展 `docs/superpowers/generation/image-gen-asset-workflow.md` 中已确定的宠物素材质量标准。该文档规定的 Codex 内置 `image_gen` 仅用于开发期项目素材；用户运行时生成由服务端接入的腾讯云图像模型完成。

## 5. 数据模型

### 5.1 自定义宠物任务

新增 `custom_pet_jobs` 集合，至少包含：

```text
jobId
ownerId
status
step
version
sourceFileId
candidateVersion
candidateSpriteFileIds
speciesLabel
mappedSkin
traits
reservedPoints
settlementStatus
rerollUsed
attemptsByStep
errorCode
errorCategory
createdAt
updatedAt
```

建议任务状态：

```text
uploaded
analyzing
generating_idle
generating_variants
validating
preview_ready
rerolling
adopted
cancelled
failed
deleting
deleted
```

任务文档只允许创建者和服务端读取。客户端不能直接修改状态、积分、资格或资源引用。

### 5.2 宠物数据

扩展 `PetData`：

```ts
type PetKind = "standard" | "custom";

type StandardPetAssetRef = {
  kind: "standard";
  skin: PetSkin;
};

type CustomPetAssetRef = {
  kind: "custom";
  templateSkin: PetSkin;
  customAssetId: string;
};
```

标准宠物继续使用现有公开远端素材。自定义宠物通过 `customAssetId` 查询资源文档，解析四状态私有 `fileID`。

自定义宠物的行为规则使用 `templateSkin`：

- 食物配置继承映射模板；
- 三档价格和饱食恢复值不变；
- 最高级专属食物继承模板；
- 游戏中需要模板分类、尺寸或动作时使用 `templateSkin`。

### 5.3 生成资格

生成资格不能只依赖可删除的任务或宠物数据。服务端维护单独的、最小化的用户资格记录：

```text
ownerId
customPetGenerationUsed
usedAt
```

成功进入首个可用预览后即视为单次生成资格已使用。后续取消领养或永久删除均不恢复资格。技术失败、输入不合格或安全拒绝不消耗成功资格。

## 6. 积分一致性

积分冻结、结算和解除全部由服务端 Node SDK 事务执行。

### 6.1 提交事务

- 检查余额至少为 300；
- 检查没有活动任务；
- 检查单次生成资格未使用；
- 写入任务；
- 增加冻结积分 300；
- 保持可用余额不能被其他消费重复使用。

### 6.2 确认领养事务

- 验证任务属于当前用户且处于 `preview_ready`；
- 验证四状态资源完整；
- 验证生成资格墓碑属于当前任务；
- 创建自定义宠物；
- 冻结积分转实际消费；
- 标记任务已领养。

重复确认请求必须返回相同结果，不能重复创建宠物或重复扣费。

### 6.3 失败与取消

- 技术失败、输入失败或安全拒绝：解除冻结，不消耗生成资格。
- 用户在有效预览后主动放弃：解除冻结，但不恢复生成资格。
- 免费重做不新增冻结金额。

## 7. 私有资源与隐私

### 7.1 存储

原图和生成图不得写入现有公开 `assets/pets/`。建议路径：

```text
users/{ownerId}/custom-pets/{jobId}/source/
users/{ownerId}/custom-pets/{jobId}/candidates/{version}/
users/{ownerId}/custom-pets/{jobId}/active/
```

云存储使用“仅创建者和管理员可读写”的私有权限。小程序通过服务端或 SDK 获取短期临时 URL，不保存永久公开 URL。

### 7.2 可见性

- 应用内只有创建者可以读取自定义宠物原图和结果图。
- 其他用户无法通过宠物 ID、文件路径或临时 URL 查询接口获取该资源。
- 临时 URL 过期后重新解析，不把签名 URL永久写入宠物数据。
- 自定义宠物可以出现在用户主动导出的静态分享截图中。
- 分享前提示截图会向他人展示该宠物形象。
- 系统不生成可公开访问的自定义宠物素材链接。

### 7.3 数据权利

上传前明确告知处理目的、保存方式和删除入口。清除账号数据时必须联动删除原图、结果图、任务和宠物数据。

## 8. AI 内容标识

- 宠物详情和结果预览显示“AI 生成”显式标识。
- 精灵图片本身不绘制角标，避免破坏透明素材和游戏展示。
- 生成文件写入 AI 生成相关元数据。
- 导出的分享截图包含可见的“AI 生成”标识。
- 任务保存模型、提示词版本和生成时间等必要审计信息，但日志不得记录原图二进制、长期临时 URL或用户宠物名称等非必要信息。

## 9. 全场景资源接入

引入统一 `PetAssetRef` 和资源解析服务，逐步替换只接受固定 `PetSkin` 的显示接口。

### 9.1 PetSprite

`PetSprite` 接收宠物资源引用或宠物实例：

- 标准宠物继续调用现有公开素材解析；
- 自定义宠物解析私有四状态 `fileID`；
- 临时 URL 使用内存与本地缓存，并根据有效期刷新；
- 加载失败保持固定尺寸占位并重试，不回退到本地图片。

### 9.2 首页与宠物小院

- 等用户数据和自定义资源引用返回后再显示宠物浮窗。
- 首次加载可预取当前自定义宠物的 `idle` 和当前状态图片。
- 返回首页时继续复用会话缓存，避免重复 loading 和图片闪烁。

### 9.3 游戏

- 宠物速数、农场进出和宠物记忆从当前用户的已领养宠物实例建立题库。
- 自定义宠物仅进入其创建者的题库。
- 题目资源在开局前解析和预加载，加载失败时从本局题库移除该实例，不泄露或回退到其他用户资源。
- 当游戏逻辑需要物种分类时使用 `templateSkin`，视觉展示使用自定义四状态资源。

## 10. 错误处理

错误分为四类：

### 10.1 用户可修复

例如图片模糊、多主体、非宠物、主体过小或格式异常。

- 解除积分冻结；
- 不消耗生成资格；
- 提示用户更换图片；
- 不创建候选宠物。

### 10.2 内容安全拒绝

- 输入或输出审核失败时停止任务；
- 解除积分冻结；
- 不消耗生成资格；
- 只显示通用安全提示，不返回审核敏感细节。

### 10.3 系统临时错误

例如超时、云服务限流、图片下载错误或内部服务错误。

- 对当前步骤进行指数退避；
- 单步骤最多自动重试 3 次；
- 重试不重复冻结积分或创建文件版本；
- 超过重试次数后进入 `failed` 并解除冻结。

### 10.4 质量失败

例如身份漂移、主体裁切、动作不可辨认或透明处理失败。

- 仅重新生成失败步骤；
- 身份锚点失败时废弃其派生状态；
- 达到重试上限后按最终失败处理。

## 11. 质量检查

自动检查至少覆盖：

- 输入中只有一个主要宠物主体；
- 四状态物种和外形身份一致；
- 毛色、花纹、体型和关键配饰一致；
- 动作状态在小尺寸下可区分；
- 图片尺寸、格式和透明通道正确；
- 四角透明且没有明显背景残留；
- 主体没有被裁切并留有安全边距；
- 没有文字、边框、可见水印或无关物体；
- 输入与输出内容安全审核通过。

自动检查不能完全替代生成效果评估。上线前应准备覆盖 7 种现有模板、范围外物种、照片输入和插画输入的固定视觉验收集。

## 12. 测试计划

### 12.1 单元测试

- 任务状态机合法转换；
- 每用户一次成功资格；
- 免费重做仅能使用一次；
- 300 积分冻结、结算和解除；
- 取消和永久删除不恢复资格；
- 食物配置继承 `templateSkin`；
- 标准与自定义资源引用解析。

### 12.2 服务集成测试

- 并发提交只能创建一个活动任务；
- 重复确认不会重复扣费或创建宠物；
- Worker 重复执行同一步骤保持幂等；
- 函数超时后定时恢复任务可继续；
- 重做失败时第一套候选仍可使用；
- 删除任务可从部分失败状态恢复；
- 私有文件只能由所属用户和服务端读取；
- 临时 URL 过期后能够刷新；
- 用户 A 无法查询、读取或删除用户 B 的任务和图片。

### 12.3 端到端测试

- 上传、后台生成、退出页面、返回恢复、预览、命名和领养；
- 输入失败、安全拒绝和系统失败后的积分状态；
- 免费重做和原子替换；
- 放弃预览；
- 永久删除与账号数据清除；
- 首页、小院、宠物速数、农场进出和宠物记忆中的显示；
- 分享截图包含 AI 标识且不暴露永久素材地址；
- 首页返回和游戏切换不重复显示全屏资源 loading。

### 12.4 视觉测试

- 现有 7 种模板各准备照片和插画样例；
- 增加鸟、仓鼠、蛇等范围外物种样例；
- 检查身份锚点和三种派生动作的一致性；
- 检查透明边缘、主体边界和小尺寸可读性；
- 检查标准宠物与自定义宠物同屏时的风格一致性。

## 13. 运维与观测

首版不根据运营指标自动限制用户，但记录以下指标：

- 提交次数；
- 输入拒绝率；
- 四状态完成率；
- 各步骤平均耗时；
- 自动重试次数；
- 免费重做率；
- 确认领养率；
- 单只宠物平均图片调用量和估算成本；
- 孤立文件和删除失败数量。

日志和告警用于发现平台故障、持续失败步骤和资源清理问题。不得在日志中输出用户原图内容或可长期访问的资源地址。

## 14. 不在首版范围

- 用户手动选择或修正模板；
- 多图参考；
- 多种视觉风格；
- 为每只自定义宠物生成专属食物；
- 同一用户生成多只自定义宠物；
- 付费购买额外生成次数；
- UID、IP 或每日总预算业务限流；
- 白名单灰度；
- 公共宠物作品广场；
- 向其他用户公开自定义宠物素材；
- 训练每只宠物的独立模型。
